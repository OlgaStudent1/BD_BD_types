### Задание 1

`Кеширование помогает решить следующий пул проблем:
1.1. Медленный отклик приложения. Без кеширования каждое обращение к данным требует запроса к базе данных или внешнему сервису, что увеличивает время ожидания для пользователя. Кеширование позволяет быстро отдавать часто запрашиваемые данные из быстрой памяти, значительно ускоряя отклик.
1.2. Перегрузка базы данных. При большом количестве одновременных пользователей база данных может стать «бутылочным горлышком». Кеширование снижает нагрузку на основную БД, забирая на себя повторяющиеся запросы.
1.3. Пиковые нагрузки и сбои при резком росте трафика. Например, при публикации резонансной новости или акции большое количество пользователей одновременно обращается к одним и тем же данным. Кеширование сглаживает такие всплески, предотвращая отказ системы.
1.4. Высокий расход ресурсов. Частые обращения к диску, сети и процессору БД увеличивают затраты на инфраструктуру. Кеширование экономит ресурсы, уменьшая количество тяжёлых операций.
1.5. Недоступность данных при сбое основного источника. Если база данных временно недоступна, кеш может отдать хотя бы устаревшие, но всё ещё полезные данные, чтобы пользователь не видел ошибку.
1.6. Хранение временных данных. Необходимость временного хранения данных, таких как сессии пользователей, токены или результаты промежуточных вычислений.
